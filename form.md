## Etapa 1
1. Otimizar o processo de gerenciamento de pessoas pelo RH.
2. Todas as empresas que tem mais de 30 funcionários já começam a ter problemas de gerenciamento no RH. Nosso foco é resolver o problema delas.
3. Funcionários de RH e donos de empresa que lidam diretamente com a folha de pagamento e dados de funcionarios.
4. Otmizar a produtividade e o controle do RH é essencial para qualquer empresa. Um RH confuso gera débito tanto pessoal quanto financeiro, por confusão de salários e até mesmo não compreender a posição de cada pessoa na equipe.
5. A todo momento setores de RH sofrem complicações derivadas de falta de organização. Imagine no pagamento de folha salárial o funcionário do RH precisar sempre olhar numa planilha enorme os dados dos funcionários e por causa disso confundir o salário de duas pessoas. ALém do desconforto nos funcionários pode acarretar em processos judiciais.

## Etapa 2
1. Construir um software de gerenciamento de pessoas pelo RH onde possa haver controle e facilidade de uso.
2. Ter um software com uma experiencia de usuário simples e funcional, com funcionaldiades que facilitem e que otimizem o trabalho do RH vai ajudar a reduzir os erros humanos causados por falta de organização e controle.
3. Funcionários de RH das empresas e Sócios.
4. Os dados dos funcionários da empresa.
5. Lista de funcionários, salários, setores da empresa, dados dos funcionários e etc...

## Etapa 4
1. Fizemos uma reunião rápida e análisamos o que é essencial para um RH e como poderiamos melhorar o fluxo de trabalho para esse setor.
2. Vamos criar uma "tela de rascunho" funcional usando Python. A ideia é fazer uma interface simples que já consiga salvar um nome e um cargo em um arquivo, para vermos como o programa se comporta antes de gastar tempo com o visual final.
3. Vamos fazer pesquisa perguntando para pessoas que trabalham nesse setor para entender como é o dia a dia das pessoas, mostrar o aplicativo e entender o que pode melhorar.
4. A ideia é que ao longo do projeto vamos ir sentindo a o fluxo de desenvolvimento e, vamos alocando as novas necessidades numa lista de prioridades.

## Etapa 5
1. Uma tela para digitar o nome, CPF e cargo do funcionário, um botão para salvar esses dados e uma lista para conseguir enxergar quem já foi cadastrado.
2. Uma aplicação web onde o usuário vai poder cadastrar dados básicos do funcionário.
3. No segundo momento, vamos adicionar busca por nome e cadastro de departamento.
4. Marcar um funcionário como demitido, cadastrar dados de conta bancária, data das próximas férias e etc...
5. A cada nova função que o programador do grupo terminar, os outros integrantes vão tentar "quebrar" o programa: digitar números onde é texto, deixar campos vazios ou cadastrar nomes repetidos para ver se o Python mostra algum erro ou se o sistema continua funcionando bem.
